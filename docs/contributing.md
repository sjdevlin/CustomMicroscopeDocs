# Contributing

## Purpose

Keep this documentation accurate, testable, and synchronized with the physical system.

## Content standard for procedure pages

Each operational page should include:

- Goal
- Scope or preconditions
- Tools/materials
- Procedure steps
- Verification criteria
- Common failures
- References

## Change rules

- Update docs in the same change as hardware/software modifications.
- Replace placeholders with measured values and evidence links.
- Prefer tables and checklists over long prose.
- Include dates, owners, and revision identifiers.

## Review checklist

- [ ] Steps can be executed by someone other than the author.
- [ ] Pass/fail criteria are objective.
- [ ] Links to datasets/scripts/photos are valid.
- [ ] Safety implications are documented.
- [ ] A change log entry is added when behavior changed.
